# Governance Evolution Map v1.0

## Purpose

This document maps the evolution of the governance ideas represented in this portfolio.

The purpose is to show how concepts moved from public research, to discussion, to implementation, to evidence.

---

# Chain 1 — Authority to Admissibility

## Origin

IPC1 — Authority Failure

Core idea:

AI systems may fail not because intelligence is missing, but because authority is undefined.

## Development

IPC3 — Externalized Governance

Core idea:

Authority must be externalized to remain stable under change.

## Transition

IPC4 — Trajectory Control

Core idea:

Governance must control behavior across transitions, not only validate isolated decisions.

## Formalization

IPC5 — Admissibility & Execution Boundary

Core idea:

A correct decision can still fail if it is not admissible at execution.

## Implementation Evidence

ARC demonstrates this chain through:

* deterministic rule evaluation
* schema validation
* explicit separation of extraction, evaluation and authority
* preserved human authority boundary

---

# Chain 2 — Documentation to Evidence

## Origin

IPC8 — Operational Momentum vs Authority

Core idea:

Operational momentum can begin to substitute formal authority signals.

## Development

IPC9 — Declared Control vs Demonstrated Control

Core idea:

The existence of a control is documentation.

The exercise of a control is evidence.

## External Validation

This chain was reinforced through governance discussion around:

* independent verification
* evidence durability
* operational governance
* documentation versus enforcement

## Implementation Evidence

CEIE supports this chain through:

* runtime evidence review
* pressure observations
* verification discipline
* audit-oriented governance documentation

---

# Chain 3 — Evaluation to Authority Boundary

## Origin

Governance boundary discussions around admissibility and accountability.

## Core Distinction

Evaluation can inform governance.

Evaluation must not become authority.

## Development

The thread evolved toward:

* proof is not governance
* evaluation is not authority
* accountability migration risk
* consequence ownership must remain external to evaluation mechanisms

## Portfolio Relevance

This chain supports the portfolio principle:

Humans retain responsibility for governance decisions and outcomes.

---

# Chain 4 — Governance Observability

## Origin

Discussion around governance declaration versus governance observability.

## Core Idea

Formal governance can remain declared while observable governance state degrades during propagation.

## Development

This chain connects:

* governance state reconstruction
* evidentiary continuity
* auditability
* runtime governance visibility

## Implementation Evidence

CEIE provides the strongest evidence base for this chain through:

* audit records
* semantic governance layers
* replay verification
* topology preservation
